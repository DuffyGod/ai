# AI发展模拟器项目开发需求

## 项目概述
基于"人生重开模拟器"的游戏机制，开发一个以AI产品为主角的模拟器网站。玩家将扮演一款AI产品，体验从规划到发展的完整生命周期。

## 游戏流程设计

### 第一阶段：准备阶段
玩家需要完成以下步骤：
1. **词条抽取**：点击"10连抽"按钮，系统随机生成10个词条
2. **词条选择**：从10个词条中选择3个作为AI的核心特性
3. **属性分配**：为5个核心属性分配10个决策点数

### 第二阶段：体验阶段
- 通过点击交互推进游戏进程
- 每回合系统基于事件库生成一条事件
- 持续进行直到触发结局事件
- 游戏结束后，玩家可选择保留一个词条，在下一局游戏中还会抽取到

## 页面架构

### 1. 落地页（主菜单）
- 网站入口页面
- 提供"开始游戏"功能
- 游戏结束后返回此页面

### 2. 准备阶段页面
**第一步：词条抽取**
- 显示"10连抽"按钮
- 点击后进入词条选择界面

**第二步：词条选择**
- 展示10个随机词条（显示名称和功能描述）
- 玩家选择3个词条确认

**第三步：属性分配**
- 展示5个核心属性面板
- 玩家分配10个决策点数
- 确认后进入体验阶段

### 3. 体验阶段页面
**页面布局**：
- 顶部信息栏：显示当前资产和属性
- 主体事件栏：按时间顺序展示已发生的事件记录
- 交互：点击任意位置生成新事件，推进回合
- 结局：触发结局事件时，详细展示结局的名称和描述
- 保留机制：展示本局的词条，可选择一个保留至下局随机池

## 核心游戏机制

### 词条系统
- **稀有度分级**：白色、蓝色、紫色、金色，按不同权重出现
- **功能体现**：在准备阶段显示名称和功能，在体验阶段影响事件发展
- **特殊效果**：部分词条具有特殊能力，可在任意环节产生影响

### 属性系统
**五大核心属性**：
1. 效率与规模
2. 认知能力
3. 可控度
4. 适应能力
5. 可解释性

**分配机制**：
- 初始属性值为随机生成，与决策点数无关
- 决策点数分配影响特定事件的走向

### 事件系统
**事件库结构**：
- 包含影响发展方向的事件和提供趣闻的无意义事件
- 事件触发受回合数、词条、资产、属性等多重条件约束
- 可能产生状态标记，影响后续事件发展
- 结局事件在特定条件下触发（高回合数、低资产、特定状态标记）

**事件生成逻辑**：
- 基于多重约束条件进行筛选
- 考虑决策点数、词条、资产、属性等影响因素
- 使用独立JSON文件维护事件库

## 开发任务清单

### 技术实现要求
1. **前端框架搭建**
   - 构建三个主要页面的前端架构
   - 确保代码可维护性和扩展性
   - 建立本地数据存储机制

2. **游戏逻辑实现**
   - 完整的游戏流程控制
   - 词条抽取和选择机制
   - 事件生成和判定系统
   - 结局触发条件判断
   - 支持游戏重复进行

3. **内容创作要求**
   - 创建20个白色稀有度词条
   - 创建10个蓝色及以上稀有度词条
   - 设计20个影响发展方向的事件
   - 设计20个提供趣闻的无意义事件
   - 内容要求：幽默风趣，取材自LLM相关新闻、缺陷、趣事和公众评价
   - 词条和事件用独立的JSON文件维护，包含ID、名称、内容，以及与游戏逻辑相关的必要信息
